\section{Problem Definition}
The operator- and user-based combined rebalancing of a shared e-scooter service can be mathematically defined as follows:
The service area is divided into zones defined by a grid system to address the issues associated with the nature of free-floating e-scooters, which can be picked up and dropped off at any position within the service area.
The use of such a grid system creates an abstraction layer for the continuous spatial space and thus reduces complexity.
For large-scale shared e-scooter services, a community structure is often employed to aggregate the service area further.

\subsubsection{Constraints}

\begin{equation}
    \sum_{z \in \mathcal{Z}}n_z(t) = \mathcal{N}, \forall t\in \mathcal{T}
\end{equation}

\begin{equation}
    x_{v, z\rightarrow z'}(t) \leq V_k, \forall z\in \mathcal{Z}, z' \in \mathcal{Z}, v \in \mathcal{V}, t \in \mathcal{T}
\end{equation}

\begin{equation}
    s_z(t) = \max\{0, P_z(t) - [n_z(t) + D_z(t)]\}
\end{equation}\label{eq:problem-definition-unmet-demand}


\[
\begin{aligned}
n_z(t+1) =\;&\ n_z(t) - P_z(t) + D_z(t) \\
&+ \sum_{v \in V} \sum_{z' \in Z} x_{v,z'\rightarrow z}(t) - \sum_{v \in V} \sum_{z' \in Z} x_{v,z\rightarrow z'}(t) \\
&+ \sum_{z' \in Z} u_{z'\rightarrow z}(t) - \sum_{z' \in Z} u_{z\rightarrow z'}(t)
\end{aligned}
\]\label{eq:problem-definition-demand-flow}

\begin{align}
\min\;\sum_{t\in T}\sum_{z\in Z} s_z(t)
\end{align}\label{eq:problem-definition-objective}

\section{Framework}
A hierarchical reinforcement learning (HRL) framework is used to tackle the rebalancing problem.
The HRL framework consists of two tiers.
The first tier is the allocator agent, which tackles the imbalances between communities.
Then, for each community, a rebalancer agent is assigned who addresses the imbalances within their community.
The allocator assigns each community a target number of vehicles.
This imbalance is then addressed by employing service workers who pick-up the desired number of vehicles at communities with oversupply and drop them off at communities with undersupply.
The second tier of this rebalancing framework comprises rebalancer agents that address the imbalances occurring within the communities.
Therefore, for each community, there is one rebalancer agent employed, which balances the vehicle distribution through user incentives.
These incentives are given to the users for dropping off vehicles at the end of their trip at different locations where they are initially intended to drop-off.


\subsubsection{Allocator Agent}
Implemented as a Multi-Head DQN (1 head for each community with the actions [-15, -10, -5, 0, 5, 10, 15] per community). Start with 8 communities. Inter-community imbalances.
\begin{itemize}
    \item \textbf{State}
    \begin{itemize}
        \item Demand forecast ($\hat{P_z}(t)$, $\hat{D_z}(t)$ $\forall z \in \mathcal{Z}$), current vehicle counts per community ($n_z(t), \forall z \in \mathcal{Z}$), 
    \end{itemize}
    \item \textbf{Action}
    \begin{itemize}
        \item assigns quotas to each of the communities (e-scooters to add or remove)
    \end{itemize}
    \item \textbf{Reward}
    \begin{itemize}
        \item Satisfied demand, negative rebalancing costs, (negative gini coefficient)
    \end{itemize}
\end{itemize}

\subsubsection{Rebalancer Agent}
Implemented with the PPO algorithm due to the large action space. Intra-community imbalances.
\begin{itemize}
    \item \textbf{State}
    \begin{itemize}
        \item Local demand forecast ($\hat{P_z}(t)$, $\hat{D_z}(t)$ $\forall z \in \mathcal{Z}_c$), local vehicle counts per grid cell($n_z(t)$ $\forall z \in \mathcal{Z}_c$)
    \end{itemize}
    \item \textbf{Action}
    \begin{itemize}
        \item Assigns price incentives for each grid cell for dropping off at these grid cells
    \end{itemize}
    \item \textbf{Reward}
    \begin{itemize}
        \item Satisfied demand ($s_z(t)$ $\forall z \in \mathcal{Z}_c$), negative incentive costs
    \end{itemize}
\end{itemize}
