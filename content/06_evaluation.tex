\section{Hyperparameter Optimization}\label{sec:evaluation-ho}
The performance of reinforcement learning agents is highly dependent on the selection of their hyperparameters.
A systematic hyperparameter optimization (HO) is therefore essential to ensure agent efficacy, particularly for a complex hierarchical reinforcement learning (HRL) framework such as the one proposed.
A dedicated HO process was conducted for both the Regional Distribution Coordinator (RDC) and the User-Incentive Coordinator (UIC) agents to maximize their individual performance.
This section outlines the optimization methodology and presents the final hyperparameter configurations used in the subsequent evaluation of the HRL framework

The optimization objective for both agents was to maximize a composite reward function.
This function is a weighted sum of three metrics: maximizing satisfied user demand ($R_{demand}$), minimizing rebalancing costs ($R_{cost}$), and promoting equitable vehicle distribution ($R_{gini}$).\\

For the HO process, the Optuna framework was utilized, a state-of-the-art optimization tool \cite{akiba_optuna_2019}.
The Tree-structured Parzen Estimator (TPE) was selected as the search algorithm due to its effectiveness in complex, high-dimensional search spaces \cite{akiba_optuna_2019}.
TPE probabilistically models the search space to efficiently identify promising hyperparameter configurations.
The algorithm fits a Gaussian Mixture Model (GMM) to a set of well-performing parameter configurations and calculates a ratio against a GMM fitted to the remaining configurations.
The sampler then selects the parameter value that maximizes this ratio, guiding the search toward more promising regions \cite{akiba_optuna_2019}.

To ensure reproducibility, all optimization runs were seeded.
As HO is computationally intensive, the process was parallelized across multiple jobs.
Furthermore, demand forecasts were pre-calculated to reduce computation time. The remainder of this section details the specific HO processes for the RDC and UIC agents.

\subsection{Regional Distribution Coordinator (RDC)}
The RDC agent, based on a Multi-head Deep Q-Network (MH-DQN) architecture with Prioritized Experience Replay (PER), has 17 distinct hyperparameters.
To manage this complexity, a three-round coarse-to-fine optimization strategy was adopted.
For all rounds, each trial consisted of 100 training episodes, each with a maximum of 100 steps, followed by 20 evaluation episodes.

\begin{enumerate}
    \item 
        In the initial round, the hyperparameters were organized into five logical groups: Learning Rate, Architecture, Replay Buffer, Exploration Strategy, and Reward Weights.
        An independent search was run for each group over 50 trials using broad search spaces to identify promising regions.
    \item 
        In the second round, the search range for each hyperparameter was refined based on the initial results.
        Ranges were expanded for parameters whose optimal values were found at the boundaries of the search space and narrowed for those found centrally, allowing for a more granular search.
        This focused round was also conducted over 50 trials per group.
    \item 
        In the final round, a unified fine-tuning sweep was performed across all 17 hyperparameters simultaneously over 100 trials.
        This approach accounts for interdependencies between parameters from different groups.
        The search spaces were further narrowed based on the results from the second round to converge on the optimal configuration.
\end{enumerate}

\subsection{User-Incentive Coordinator (UIC)}
The UIC agent, which employs a Proximal Policy Optimization (PPO) algorithm, has 15 distinct hyperparameters to be optimized.
A multi-stage optimization strategy, similar to that used for the RDC, was implemented over three rounds.
Each trial in every round was run for 50,000 timesteps.

\begin{enumerate}
    \item 
        The initial round identified promising search areas using broad hyperparameter ranges.
        For the first two rounds, the parameters were divided into four logical groups: PPO Core, Architecture, Stability, and Reward Weights.
        The initial round consisted of 30 trials.
    \item 
        The second round refined the search by narrowing or expanding the parameter ranges based on the initial outcomes, following the same logic as the RDC optimization.
        The second round again consisted of 30 trials.
    \item 
        The final round optimized all 15 parameters simultaneously to fine-tune the configuration, thereby accounting for any cross-group parameter dependencies.
        To account for the increase in complexity, the final round consisted of 100 trials.
\end{enumerate}

\section{Experimental Setup}\label{sec:evaluation-setup}
Fleet sizes have been determined by the mean number of vehicles available based on all GBFS snapshots.
For training and hyperparameter optimization, this fleet size has been reduced to 90\% to allow the agent to learn from more experiences with undersupply.
For the evaluation, three different fleet sizes were used: the mean from the GBFS data, 80\% of the mean, and 120\% of the mean.
Evaluating the models with different fleet sizes results in a more robust evaluation since, multiple scenarios are tested, including scenarios with oversupply and with undersupply.

\subsection{Baselines}\label{sec:evaluation-setup-baselines}
For evaluating the proposed HRL rebalancing framework, multiple baselines have been chosen.
The evaluation against multiple baselines elevates the interpretability of the results.
Each baseline has been chosen to test a different hypothesis, which should test the design choices of the proposed framework thoroughly.

\subsubsection{RDC-only Baseline}
The first baseline for comparison is running the Regional Distribution Coordinator (RDC) only without the User-Incentive Coordinators (UICs).
By comparing the results to this baseline, it can be observed whether broad operator-based rebalancing is enough to increase trip satisfaction.


\subsubsection{UIC-only Baseline}
The second baseline is running the second tier of the proposed HRL framework only, i.e., only the UIC agents propose incentive levels to locally rebalance the system without any rebalancing between communities and focusing on user-based rebalancing.
With this baseline, the hypothesis is whether local fleet optimization can result in similar results as if the system is rebalanced over multiple hierarchies.

\subsubsection{Greedy Heuristic}
The third baseline implemented as a greedy heuristic should test whether a hierarchical Reinforcement Learning approach can outperform simple rule-based algorithms.
This baseline serves as a lower bound that the proposed HRL framework should outperform.

This greedy heuristic is implemented in a similar two-tier hierarchy mimicking the design of the proposed HRL framework.
The top-tier greedy heuristic proposes rebalancing quotas for manual rebalancing, similarly to the RDC.
The greedy strategy works as follows: first, for each community, the expected deficit of vehicles is calculated based on the current vehicle counts and the demand forecasts provided by the Demand Forecasting Module.
This list is then sorted in descending order by the calculated deficit to identify the communities with the highest gap between supply and expected demand.
The rebalancer iterates over this prioritized list and determines the number of vehicles that ideally would be needed to add.
It then selects the most suitable action value from the same discrete action values used by the RDC.
The same is done for communities with an expected surplus of vehicles.
Here, action values are selected to match supply and expected demand by removing vehicles from these communities.
Based on the maximum number of rebalanced vehicles, the selected action values for each community with the most surplus or deficit are chosen.
Additionally, the action values are scaled to meet fleet size preservation and only non-negative vehicle counts per community.

The second tier of this greedy heuristic works similarly to the UIC agents.
This heuristic is applied to each community and determines incentive levels for each zone within the community to encourage users to drop-off vehicles in zones that are predicted to have a vehicle deficit.
First, the ratio between supply and expected demand is calculated for each zone.
A threshold (e.g., 0.8) is defined by which the maximum incentive is given for dropping off a vehicle at that zone.
If the ratio for a zone exceeds the threshold or no demand is expected, then no incentive is offered.
